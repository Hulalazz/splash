What is Stochastic Algorithm

Stochastic algorithm is a family of algorithms which processes a large dataset by sequentially processing its random samples. Since their per-iteration computation cost is independent of the data size, these algorithms are very efficient in learning from large-scale data. Typical examples of stochastic algorithm include:

  • Stochastic Gradient Descent (SGD)
  • Stochastic Dual Coordinate Ascent (SDCA)
  • Markov chain Monte Carlo (MCMC)
  • Gibbs Sampling
  • Stochastic Varitional Inference
  • Expectation Propagation.

Traditionally, stochastic algorithms are difficult to parallelize due to their sequential nature. To run these algorithms on a distributed system, there are three questions to ask:

  • How to speedup a sequential algorithm with (inconsistent) parallel updates?
  • How to minimize the communication cost between nodes?
  • How to design a user-friendly programming interface for Spark users?

Our simple answer is Splash.


What is Splash

Splash is a general-purpose framework for parallelizing stochastic algorithms on multi-node clusters. You can develop any stochastic algorithm using the Splash programming interface without knowing any detail about distributed computing. The parallelization is automatic and is communication efficient. Splash is built on Scala and Apache Spark, so that you can employ it to process the Resilient Distributed Datasets (RDD).

On large-scale datasets, Splash can be substantially faster than existing data analytics packages based on Apache Spark. For example, to learn a 10-class logistic regression model on the mnist8m dataset, The SGD algoritihm implemented via Splash is 25x faster than MLlib’s L-BFGS and 75x faster than MLlib’s mini-batch SGD for achieving the same logistic loss. All algorithms run on a 64-core cluster.
